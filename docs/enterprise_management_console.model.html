Detailed Submenu Structure

1) Database Management — The technical foundation of the app, covering the full lifecycle of database administration.
   • Connections — Manage connection strings, credentials, and test connectivity for MySQL, Postgres, SQL Server, and Oracle. (Already developed)
   • Backup & Restore — Schedule automated backups, manage backup storage (local, S3), and perform point-in-time restores.
   • Performance Monitoring — Dashboard for query performance, CPU/memory usage, active connections, and slow query logs.
   • Query Runner — Integrated SQL editor to run ad-hoc queries.
   • Replication & Clustering — Configure and monitor replication for high availability and load balancing.

2) Tables & Views — Focuses on data architecture and schema manipulation.
   • Schema Browser — Visual explorer for tables, views, columns, indexes, and data types. (Already developed as “Table List”)
   • Table Designer — Visual tool for creating and modifying table structures. (Already developed as “Create Table”)
   • View Designer — UI for building and managing SQL views without writing raw code.
   • Data Import/Export — Map and import files (CSV, Excel, JSON), and export table data. (Partially developed)
   • Schema Migration — Track schema changes over time, with versioning and rollback options.

3) File Management — Bridges raw data files with structured databases.
   • File Explorer — Manage files on server or storage (local, FTP, S3). (Already developed)
   • ETL Jobs (Extract, Transform, Load) — Automate file ingestion, parsing, transformation, and database loading.
   • File Parsers — Define templates for parsing file formats (fixed-width, delimited, JSON, XML).
   • Storage Connectors — Configure cloud storage (Amazon S3, Google Cloud Storage).

4) Report Management — The BI and analytics hub.
   • Report Dashboard — Central access to all generated reports. (Already developed)
   • Report Designer — Drag-and-drop builder for reports, charts, and graphs.
   • Data Sources — Manage reportable data sources (e.g., Sales, Inventory).
   • Scheduled Reports — Automate report generation and email delivery.
   • Analytics Workbench — Advanced pivot, filter, and drill-down tools.

5) Artificial Intelligence Access — Unlocks predictive insights and automation.
   • Model Training — Workspace for training ML models (sales forecasting, churn analysis).
   • Predictive Analytics — Apply trained models to new data for actionable insights.
   • AI-Powered Alerts — Configure anomaly detection alerts (e.g., sales drops).
   • Natural Language Query — Ask questions in plain English (e.g., “Top 10 products last month?”).

6) Communications — Handles both internal and external communication.
   • Email & SMS Gateway — Configure gateways for transactional emails and SMS alerts.
   • Internal Chat/Messaging — Messaging tool for console users.
   • Notification Templates — Pre-define system notification formats.
   • API Webhooks — Manage inbound/outbound hooks for integrations (Shopify, payment gateways).

7) Preferences & Settings — Administrative control center.
   • User & Role Management — Define users, roles, and granular permissions.
   • System Configuration — General system settings (themes, languages, security).
   • Audit Log — Searchable log of all user/system actions.
   • Application Builder — Low-code tool to scaffold mini-apps (forms, tables, views).
   • Licensing & Billing — Manage enterprise licenses and track usage for billing.

Implementation Notes

• React + Tailwind: Use shadcn/ui and lucide-react for consistent UI primitives and icons.

• State & Routing: Prefer React Router (standalone SPA) or Inertia (Laravel + Vite) for seamless page transitions.

• Charts: Recharts for time-series/aggregations in Performance Monitoring and Report Designer.

• Access Control: Gate routes by role (Admin / Accountant / Warehouse / Auditor) and feature-flags by tenant.

• DX: Prettier + ESLint + TypeScript strict mode; Vitest/Playwright for tests; commit hooks via lint-staged.

function Sidebar({ openGroups, setOpenGroups, active, setActive }) {
  const Group = ({ title, icon: Icon, items }) => {
    const isOpen = openGroups[title];
    return (
      <div className="mb-1">
        <button onClick={() => setOpenGroups({ ...openGroups, [title]: !isOpen })}
                className="flex w-full items-center justify-between rounded-xl px-3 py-2 hover:bg-neutral-100 dark:hover:bg-neutral-800">
          <span className="flex items-center gap-2">
            <Icon className="h-4 w-4" />
            <span className="text-sm font-medium">{title}</span>
          </span>
          {isOpen ? <ChevronDown className="h-4 w-4" /> : <ChevronRight className="h-4 w-4" />}
        </button>
        {isOpen && (
          <div className="ml-6 mt-1 flex flex-col gap-1">
            {items.map(({ key, icon: ItemIcon }) => {
              const activeHere = active.sub === key && active.group === title;
              return (
                <button key={key} onClick={() => setActive({ group: title, sub: key })}
                        className={`flex items-center gap-2 rounded-xl px-3 py-2 text-sm hover:bg-neutral-100 dark:hover:bg-neutral-800 ${activeHere ? "bg-neutral-100 dark:bg-neutral-800 font-medium" : ""}`}>
                  <ItemIcon className="h-4 w-4" />
                  <span>{key}</span>
                </button>
              );
            })}
          </div>
        )}
      </div>
    );
  };

  return (
    <aside className="h-full w-72 shrink-0 border-r border-neutral-200 dark:border-neutral-800 p-3 overflow-y-auto">
      {Object.entries(MODULES).map(([title, def]) => (
        <Group key={title} title={title} icon={def.icon} items={def.items} />
      ))}
    </aside>
  );
}

function MainContent({ active }) {
  const key = `${active.group}:${active.sub}`;
  if (key === "Database Management:Connections") return <ConnectionsView />;
  if (key === "Database Management:Performance Monitoring") return <MonitoringView />;
  if (key === "Database Management:Query Runner") return <QueryRunnerView />;
  if (key === "Report Management:Report Designer") return <ReportDesignerView />;
  if (key === "Artificial Intelligence Access:Natural Language Query") return <NLQView />;
  return <PlaceholderView group={active.group} sub={active.sub} />;
}

export default function EnterpriseConsolePrototype() {
  const [theme, setTheme] = useState("light");
  const [openGroups, setOpenGroups] = useState(() => {
    const initial = {};
    Object.keys(MODULES).forEach((t) => (initial[t] = t === "Database Management"));
    return initial;

  const firstGroup = Object.keys(MODULES)[0];
  const firstSub = MODULES[firstGroup].items[0].key;
  const [active, setActive] = useState({ group: firstGroup, sub: firstSub });

  const themeClasses = useMemo(() => (theme === "dark" ? "dark bg-neutral-950 text-white" : "bg-neutral-50 text-neutral-900"), [theme]);
